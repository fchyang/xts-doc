# XTS DOC

An AndroidTV XTS documentation site built with Docusaurus. If you encounter XTS issues, hope this document can help you.

**Live Site:** [https://xtsdoc.cloud-ip.cc](https://xtsdoc.cloud-ip.cc)

## 📋 Overview

This is a comprehensive documentation platform for XTS (Google's Compatibility Test Suite) related topics. The project uses modern web technologies including Docusaurus 3, TypeScript, React, and MDX to provide an interactive and searchable documentation experience.

## 🛠️ Tech Stack

- **Docusaurus 3.10** - Static site generator for documentation
- **TypeScript (67.7%)** - Strongly typed JavaScript
- **MDX (22.3%)** - Markdown with JSX components
- **CSS (10%)** - Styling
- **React 19** - UI library
- **Node.js ≥20** - Runtime environment

## 📚 Documentation

The documentation covers XTS-related topics and troubleshooting. Navigate through the Tutorial section in the navbar to explore content.

### Writing Docs

Create new documentation files with these steps:

1. **Create a new file** in the `docs/` directory with `.md` or `.mdx` extension
2. **Add frontmatter** at the top of your file:
   Markdown
   
   ```
   ---
   sidebar_position: 1
   title: Your Page Title
   description: Brief description of this page
   ---
   ```
3. **Update sidebar navigation** in `sidebars.ts`:
   TypeScript
   
   ```
   const sidebars = {
     tutorialSidebar: [
       {
         type: 'autogenerated',
         dirName: 'docs',
       },
     ],
   };
   ```
4. **Use MDX features** to enhance your documentation:
   
   * Embed React components
   * Use code blocks with syntax highlighting
   * Add interactive content

Example `.mdx` file:

Markdown

```
---
sidebar_position: 2
title: Getting Started
---

# Getting Started with XTS

This is a regular paragraph, and the following is a code block.

\```jsx
function HelloWorld() {
  return <h1>Hello, World!</h1>;
}
\```

You can also embed React components directly in MDX!
```

## 🔍 Search

The site includes full-text search functionality powered by `@easyops-cn/docusaurus-search-local`:

* **Multilingual Support:** Indexes content in English and Chinese
* **Smart Indexing:** Automatically indexes all documentation pages
* **Highlight Results:** Search terms are highlighted on the result page
* **Hashed Index:** Uses hashed indexing for better performance

The search is available in the top navigation bar. Users can quickly find relevant documentation without navigating through the sidebar.

## 🌐 Deployment

### GitHub Pages Deployment

The site is configured for automatic GitHub Pages deployment:

bash

```
npm run deploy
```

This command builds the site and deploys it to the `gh-pages` branch.

### Custom Domain

The site uses a custom domain configured in the `CNAME` file:

* **Domain:** `xtsdoc.cloud-ip.cc`
* **Configuration:** Both root `CNAME` and `static/CNAME` are set up
* **DNS:** Point your domain DNS records to GitHub Pages servers

### Build and Deploy Manually

bash

```
# Build the site
npm run build

# The build output will be in the build/ directory
# You can then deploy this to any static hosting service
```

## ⚙️ Configuration

### Key Configuration Files

1. **docusaurus.config.ts** - Main site configuration
   * Site title: "AndroidTV XTS DOC"
   * Base URL and domain settings
   * Theme configuration (colors, fonts)
   * Plugin settings (search, etc.)
   * Navbar and footer structure
2. **sidebars.ts** - Documentation structure
   * Defines the left sidebar navigation
   * Controls the order of documentation pages
   * Supports nested categories and doc groups
3. **tsconfig.json** - TypeScript settings
   * Compiler options for the project
   * Path aliases for imports

### Customization Examples

**Update site title and tagline** in `docusaurus.config.ts`:

TypeScript

```
const config: Config = {
  title: 'Your New Title',
  tagline: 'Your new tagline',
  // ... rest of config
};
```

**Change theme colors** in the `themeConfig`:

TypeScript

```
themeConfig: {
  colorMode: {
    respectPrefersColorScheme: true,
  },
  // Customize Prism theme for code blocks
  prism: {
    theme: prismThemes.github,
    darkTheme: prismThemes.dracula,
  },
}
```
